UX Audit Summary – Proxy Voting Dashboard Experience

This audit explored the current limitations and future possibilities of improving the user experience across two interconnected proxy voting solutions. The primary goal was to reimagine a unified, intelligent dashboard system that caters to different user roles, enhances productivity, and supports seamless navigation between tasks and applications.

Current State Analysis

  • One platform lacked a dashboard entirely; the other had a static layout with four preset widgets and no customization.
  • Dashboards operated in isolation, forcing users to context-switch between systems for task management.
  • No mechanisms existed for highlighting deadlines, recent activity, or personal priorities.
  • Users relied on manual tracking, increasing the risk of missed tasks and compliance issues.

Audit Goals

  • Create an intuitive, proactive dashboard beyond static widgets.
  • Enable role-aware, task-driven daily command centers.
  • Support seamless workflows across systems.

Key Design Opportunities & Recommendations

1. Role-Based Personalization

  • Customize dashboard views per user type (e.g., proxy specialist, compliance officer).
  • Examples: A proxy specialist sees upcoming votes; a compliance officer sees audit alerts.

2. Configurable Widgets

  • Users can show/hide/reorder widgets like "My Alerts" or "My Reports."
  • Support different layouts based on user goals.

3. Smart Task Suggestions

  • Use behavioral data to suggest timely actions.
  • Example: "3 votes due this week — block focus time?"

4. Notification Center

  • Filter by type, urgency, client, or role.
  • Include mobile alerts and mute options.

5. Recent Activity Resume

  • Show last 2–3 items viewed or edited to streamline task resumption.

6. Time-Based and Calendar Views

  • Plan tasks via daily/weekly/deadline modes.
  • Enable focus blocks and structured planning.

7. Custom Tagging

  • Tag meetings or tasks with labels like “Urgent” or “Needs Review.”
  • Useful for handoffs and team coordination.

8. Integrated Help & Guidance

  • Contextual onboarding, micro-lessons, and support nudges.
  • Trigger help when friction is detected (e.g., errors, long pauses).

9. Change Management Support

  • “What’s New” section for updates and policy changes.
  • Provide empathetic and actionable messaging.

10. Remote Notifications

  • Opt-in mobile alerts tied to time zones and calendar data.

The Vision: A Unified Super Dashboard

The ideal future state is a centralized, intelligent dashboard that consolidates critical information from multiple systems and acts as a personalized control center for all users.

Key Features:
  • Deadlines: Upcoming votes, reports, and meetings.
  • Action Items: Real-time task updates with urgency.
  • Alerts: Unresolved issues, compliance exceptions.
  • Planner View: Personalized focus suggestions.
  • Deep Links: Direct access to relevant content.

Best Practices Incorporated

  • User Role Mapping: Designed for analysts, compliance officers, and executives.
  • Clarity: Prioritized actionable insights and real-time data.
  • Accessibility: WCAG-compliant colors, fonts, and UI patterns.
  • Scalability: Flexible enough for large portfolios and workflows.
  • Customization: Personalized without rigid templates.

Strategic Outcomes

  • Reduce missed votes and compliance risks.
  • Increase user confidence and clarity.
  • Improve task traceability and planning.
  • Support long-term scale and cross-functional collaboration.
